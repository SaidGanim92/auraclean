# AURA CLEAN — אתר חנות מוצרי ניקיון (עברית + ערבית)

אתר **סטטי** (HTML + CSS + JavaScript טהור, ללא build) לחנות מוצרי ניקיון במאגר.
דו-לשוני (עברית + ערבית, RTL), הזמנות נסגרות ב**וואטסאפ** (אין סליקת אשראי באתר).

---

## 1. הרצה מקומית (לבדיקה)

מכיוון שהאתר טוען נתונים דרך `fetch`, יש להריץ שרת מקומי (לא לפתוח את הקובץ ישירות):

```bash
cd "aura clean web site"
python3 -m http.server 8000
```

ואז לפתוח בדפדפן: `http://localhost:8000`

> ללא הגדרת גיליון Google Sheets, האתר עולה עם **קטלוג הגיבוי המובנה** (18 מוצרי הדגמה).

## 3. הקמת גיליון Google Sheets (מקור המוצרים)

1. צרו גיליון חדש ב-Google Sheets, ותנו לטאב (הגיליון התחתון) את השם `products`.
2. בשורה הראשונה הזינו את **כותרות העמודות** בדיוק כך (באנגלית):

   | id | name_he | name_ar | category_he | category_ar | price | desc_he | desc_ar | image | available | unit | on_sale | sale_price | featured |
   |----|---------|---------|-------------|-------------|-------|---------|---------|-------|-----------|------|---------|------------|----------|

   - **id** — מזהה ייחודי לכל מוצר (למשל `p01`). חובה.
   - **name_he / name_ar** — שם המוצר בעברית ובערבית.
   - **category_he / category_ar** — שם הקטגוריה (המוצרים מקובצים לפיה).
   - **price** — מחיר רגיל במספרים (למשל `18`).
   - **desc_he / desc_ar** — תיאור קצר בשתי השפות.
   - **image** — כתובת URL מלאה לתמונה, **או** שם קובץ שנמצא בתיקיית `images/` (למשל `soap.jpg`). ריק = מוצג רקע צבעוני אוטומטי עם שם המוצר.
   - **available** — `TRUE` אם במלאי, `FALSE` אם אזל (המוצר יעומעם והכפתור יושבת).
   - **unit** — יחידת מידה להצגה (למשל `לליטר`, `לאריזה`).
   - **on_sale** — `TRUE` להצגת תגית "מבצע" ומחיר מוזל.
   - **sale_price** — המחיר במבצע (רלוונטי רק אם `on_sale=TRUE`).
   - **featured** — `TRUE` כדי שהמוצר יופיע ב"מוצרים מומלצים" בדף הבית.

3. **פרסום לצפייה:** תפריט `שיתוף` → הרשאה **"כל מי שיש לו את הקישור — צופה"**.
   (לחלופין: `קובץ → שיתוף → פרסום באינטרנט`.)
4. העתיקו את **מזהה הגיליון** מכתובת ה-URL — החלק שבין `/d/` ל-`/edit`:
   `https://docs.google.com/spreadsheets/d/`**`1AbCdEf... ← זה ה-SHEET_ID`**`/edit`
5. הדביקו אותו בקובץ `js/data.js`, בתוך `CONFIG.SHEET_ID`.

> עריכת מוצרים/מלאי/מחירים נעשית ישירות בגיליון — האתר נטען מחדש ומציג את העדכון.
> אם הגיליון לא נטען מסיבה כלשהי, האתר נופל אוטומטית לקטלוג הגיבוי (הודעה בקונסול בלבד).

## 4. מילוי ה-Placeholders (חובה לפני עלייה לאוויר)

### `js/data.js` — בראש הקובץ, בבלוק `CONFIG`
- `SHEET_ID` — מזהה הגיליון (סעיף 3). כרגע: `[SHEET_ID]`.
- `WHATSAPP_NUMBER` — מספר וואטסאפ בפורמט בינלאומי ללא `+`. כבר מוגדר: `972502073111`.
- `SHIPPING_FEE` — דמי משלוח. כבר מוגדר: `20`.

### `js/consent.js` — בראש הקובץ
- `META_PIXEL_ID` — מזהה ה-Meta Pixel. כרגע: `[META_PIXEL_ID]`.
  אם לא ממלאים — ה-Pixel פשוט לא ייטען (האתר יעבוד כרגיל).

### עמודים משפטיים (`pages/*.html`)
חפשו טקסטים המסומנים בצהוב עם סוגריים מרובעים `[...]` ומלאו:
- **שם העסק** (עברית/ערבית), **עוסק/ח.פ**, **כתובת מלאה במאגר**, **מחוז** שיפוט,
  **מספר ימי עסקים** לאספקה, **שעות פעילות**, **תאריך עדכון**.
- **רכז נגישות** (בעמוד הצהרת נגישות): שם ופרטי קשר — **חובה למלא**.
- פרטי הטלפון (050-2073111) והאימייל (sagroup050@gmail.com) כבר מוזנים.

## 5. פריסה ל-Netlify

**דרך הכי פשוטה (גרירה):**
1. היכנסו ל-[app.netlify.com](https://app.netlify.com) → `Add new site` → `Deploy manually`.
2. גררו את **כל התיקייה** של האתר לחלון. תוך שניות מתקבלת כתובת אתר חיה.

**דרך Git (אופציונלי):** חברו מאגר Git ל-Netlify; אין צורך ב-build command (אתר סטטי) — Publish directory = תיקיית השורש.

לאחר הפריסה אפשר לחבר דומיין משלכם דרך `Domain settings`.

## 6. נגישות ופרטיות — נקודות חשובות

- **נגישות:** האתר בנוי לפי ת"י 5568 / WCAG רמה AA (מבנה סמנטי, ניווט מקלדת, ניגודיות,
  הגדלת טקסט, תפריט נגישות). יש **למלא את פרטי רכז הנגישות** בהצהרת הנגישות.
- **עוגיות ו-Pixel (תיקון 13):** ה-Meta Pixel וכלי מעקב **אינם נטענים** עד שהמשתמש לוחץ
  "אישור" בבאנר העוגיות. אפשר לשנות בחירה בכל עת דרך "הגדרות עוגיות" ב-footer.
- **הבהרה משפטית:** הנוסחים הם תבנית לפי דרישות החוק בישראל ואינם ייעוץ משפטי.
  מומלץ שעו"ד יעבור עליהם (במיוחד בשל ה-Pixel) לפני פרסום.
